Gen AI Training Modules

Introduction

The Gen AI Starter resources provide comprehensive training materials for a multi-day workshop focused on Generative AI and its application in Agile-driven business transformations. Led by trainer Chirantan Lonkar, the sessions are designed to give participants a strong technical and conceptual foundation in modern artificial intelligence systems, particularly Large Language Models (LLMs). The workshop begins with an introduction to the core principles behind LLMs, including tokenisation, embedding layers, and semantic similarity, helping participants understand how AI models interpret language, represent meaning, and generate responses based on contextual understanding.

A major focus of the training is Retrieval-Augmented Generation (RAG), an architecture that combines language models with external knowledge sources to produce more accurate and reliable outputs. The sessions explain how RAG helps mitigate model hallucinations by retrieving relevant information from external databases and grounding AI responses in factual data. Through practical demonstrations, participants learn how to implement AI-powered solutions such as agentic workflows, automated document processing pipelines, and intelligent search systems using modern frameworks like LangChain and vector databases.

The workshop also explores several real-world enterprise use cases where Generative AI can significantly enhance productivity and innovation. These include modernising legacy databases, automating code reviews, generating technical documentation, and improving project management through AI-driven insights and automation. By examining these applications, participants gain a clear understanding of how organisations can integrate generative technologies into existing workflows to accelerate digital transformation.

Overall, the sessions highlight the shift from traditional deterministic machine learning approaches toward non-deterministic generative systems that are capable of processing large volumes of unstructured data. By leveraging these technologies, organisations can build intelligent solutions that enhance decision-making, automate complex tasks, and significantly speed up software development and business operations.

Prompting Techniques

Refer to the complete training document provided here: Prompting Techniques Document.

This module introduces key prompting techniques and frameworks that enable effective interaction with Generative AI systems. These techniques are designed to improve clarity, structure, and accuracy of AI-generated outputs, especially when dealing with complex analytical or business problems. By applying structured prompting approaches, users can guide AI models to produce more relevant, reliable, and context-aware responses.

The TRACE framework, which stands for Task, Role, Audience, Constraints, and Elaboration, focuses on structuring prompts in a comprehensive manner. It ensures that the AI clearly understands the objective, the perspective it should adopt, the intended audience, and any limitations or specific details required. This approach is particularly useful when presenting insights or communicating complex trade-offs in business scenarios.

The CARE framework, consisting of Challenge, Action, Result, and Elaboration, is primarily used for analytical and problem-solving tasks. It helps in breaking down complex issues into structured steps, enabling the AI to generate actionable outputs such as scripts, reports, or root cause analyses. This makes it highly effective in data-driven environments where clarity and precision are essential.

The CRITIC framework is an iterative approach that allows users to refine and improve outputs by engaging the AI as a collaborator. It involves critiquing an initial idea, revising it, improving the solution, testing it, and comparing results. This method is especially useful for enhancing visualizations, reports, or strategies to make them more impactful and decision-oriented.

The APE framework, which stands for Abstract, Package, and Explain, is designed to transform one-time solutions into reusable tools. It enables users to generalize logic, package it into reusable formats such as functions, and provide clear documentation so that others can easily use and scale the solution within a team or organization.

Zero-shot prompting is a technique where the AI is given a task without any prior examples. The effectiveness of this method depends entirely on how clearly and precisely the instructions are defined. It is particularly useful in situations where quick insights are required from new or unfamiliar data without the availability of sample outputs.

Few-shot prompting improves performance by providing a small number of examples that demonstrate the expected input-output pattern. These examples help the AI better understand context, intent, and structure, making it especially effective for tasks such as classification, summarization, and pattern recognition in unstructured data.

Chain-of-thought prompting enhances reasoning by encouraging the AI to think step-by-step before arriving at a final answer. This approach is particularly beneficial for solving complex problems involving calculations, logic, or multi-step decision-making, as it increases both accuracy and transparency in the results.

ReAct prompting combines reasoning with action by allowing the AI to interact with external tools and data sources. It follows a structured cycle of thought, action, and observation, enabling the model to gather real-time information, validate its reasoning, and produce more grounded and reliable outputs.

Meta prompting is a higher-level technique that focuses on generating better prompts rather than direct answers. It involves guiding the AI to act as a prompt designer, helping users convert vague or ambiguous requirements into structured, effective prompts that can be used for further analysis or problem-solving.

Github Copilot Commands

Refer to the complete document provided here: Github Copilot Commands Reference

GitHub Copilot provides powerful commands to assist developers in code generation, debugging, documentation, and workflow automation directly within IDEs and CLI environments.

Core slash commands such as /explain, /fix, /tests, and /doc help in understanding code, fixing errors, generating unit tests, and adding documentation. These commands can be used on selected code or entire files to improve productivity.

Project and workflow commands like /new and /newNotebook allow quick scaffolding of applications and notebooks, while commands like /clear and /help assist in managing chat sessions and discovering features.

Copilot also supports contextual querying using participants such as @workspace, @vscode, @terminal, and @github, enabling users to ask questions about codebases, tools, terminal commands, and repositories.

Context variables like #file, #selection, #editor, and #codebase allow attaching relevant code or workspace data to prompts, resulting in more accurate and context-aware responses.

In the CLI environment, Copilot offers commands for session management, directory access, and configuration, along with input modes such as slash commands, shell execution, file references, and natural language prompts.

Keyboard shortcuts further enhance productivity by enabling quick access to chat, inline suggestions, and command execution within development environments.
